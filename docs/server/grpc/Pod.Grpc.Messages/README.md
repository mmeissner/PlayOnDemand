# Pod.Grpc.Messages

> The DTO-only slice of the protocol: links the message classes generated by `Pod.Grpc.Base` and decorates them with C# partial classes that add Guid/TimeSpan/DateTime conversion helpers.

## Purpose

`Pod.Grpc.Base` generates raw protobuf message classes into its `Export/` folder, but those classes use awkward primitives on the wire — `bytes` for Guids, `int64` for ticks, etc. This project:

1. **Links** the four `Messages*.cs` files from `Pod.Grpc.Base/Export/` into its own assembly (`<Compile Include="..\Pod.Grpc.Base\Export\MessagesShared.cs" Link="..." />`).
2. **Extends** them with `partial class` definitions in `Shared/Converter.cs` so callers can write `request.ConnectionId.ToGuid()` instead of `new Guid(request.ConnectionId.Value.ToByteArray())`.
3. **Provides** static `ConverterExtensions` (e.g. `Guid.ToGuidAsBytes()`, `TimeSpan.ToTimeSpanAsLong()`) so server code can build outgoing messages without touching `ByteString` / `Ticks` directly.

It deliberately excludes the **service** stubs (`Service*Grpc.cs` files) — those go to the net10.0 server projects under `Pod.Grpc.{Base,ConnectHost,ShellHost}.Server`. This split is what allows `Pod.Grpc.Messages` to stay on `netstandard2.0`, which is the only way the .NET Framework 4.7.1 kiosk can share message types with the net10.0 server.

  - `MessagesShellHost.cs` → `ShellHost/`

## Responsibility

**IS** responsible for:
- Compiling the protobuf message DTOs into a netstandard assembly that both client and server can reference.
- Hosting the `partial` extensions that add ergonomic Guid/Time conversions.
- Providing extension methods on system primitives to **build** the wire DTOs (`Guid.ToGuidAsBytes()` etc.).

**IS NOT** responsible for:
- gRPC service base classes (those are in `Pod.Grpc.ConnectHost.Server` / `Pod.Grpc.ShellHost.Server`, which link `Service*.cs` instead).
- Channel/connection plumbing (`Pod.Grpc.Base.Client`/`Server`).
- Mapping wire DTOs to domain entities (that lives in `Pod.Services` and `LeapVR.Shell.Services`).

## Public API surface

All extensions live in `Pod.Grpc.Messages.Shared`.

### `partial class GuidAsBytes`
- `Guid? ToGuidNullable()` — returns `null` if the byte payload is empty.
- `Guid ToGuid()` — unconditional convert.

### `partial class TimeSpanAsLong`
- `TimeSpan? ToTimeSpanNullable()` — returns `null` when `HasValue == false`.
- `bool ToTimeSpan(out TimeSpan timeSpan)` — Try-pattern variant.

### `partial class DateTimeUtcAsLong`
- `DateTime? ToDateTimeUtcNullable()`
- `DateTime? ToDateTimeUtcNullable(TimeSpan timeskew)` — adds clock-skew adjustment.
- `DateTime ToDateTimeUtc(TimeSpan timeskew)` — non-nullable, returns `DateTime.MinValue` if not set.
- `DateTime ToDateTimeUtc()`
- `bool ToDateTimeUtc(out DateTime dateTimeUtc)` — Try-pattern variant.

### `static class ConverterExtensions`
- `GuidAsBytes ToGuidAsBytes(this Guid? guid)` — `null` becomes `Guid.Empty` bytes (not empty bytes).
- `GuidAsBytes ToGuidAsBytes(this Guid guid)`
- `TimeSpanAsLong ToTimeSpanAsLong(this TimeSpan? timeSpan)`
- `TimeSpanAsLong ToTimeSpanAsLong(this TimeSpan timeSpan)`
- `DateTimeUtcAsLong ToDateTimeUtcAsLong(this DateTime? dateTime)` — calls `.ToUniversalTime()` first.
- `DateTimeUtcAsLong ToDateTimeUtcAsLong(this DateTime dateTime)` — calls `.ToUniversalTime()` first.

Plus all the message classes themselves — see `docs/server/grpc/Pod.Grpc.Base/README.md` for the full message catalog.

## Notable patterns / gotchas

- **`ToGuidAsBytes(Guid?)` writes `Guid.Empty` for null**, not an empty `ByteString`. This is intentional and matches the `GuidAsBytes.ToGuidNullable()` test (`if (Value.IsEmpty) return null;`) which works because `Guid.Empty.ToByteArray()` happens to be 16 zero bytes that `ByteString.IsEmpty` still treats as non-empty — meaning round-tripping `(Guid?)null` will give you back `Guid.Empty`, not `null`. If you need true null semantics use the `HasConnectionId(out var)` extension in `Pod.Services` instead (see `ShellHostServiceGrpc.GetNotifications`).
- **All datetimes are UTC ticks.** The `ToDateTimeUtcAsLong` extensions force `.ToUniversalTime()` so passing a `DateTime` with `Kind=Local` is silently corrected. The decode side stamps `DateTimeKind.Utc` on the output.
- **`timeskew`-aware decode** exists for the kiosk to compensate for clock drift between station and server. See `ShellHostServiceGrpc.GetServerSettings` which sends `ServerTimeUtcNow` exactly so the kiosk can compute its skew.
- **Nothing about this project is generated.** Only the four linked files are. `Converter.cs` is hand-written and safe to extend.

## Consumers

- `Pod.Grpc.Base.Server` (transitively) — server-side glue.
- `Pod.Grpc.ConnectHost.Server` and `Pod.Grpc.ShellHost.Server` — service implementations build response messages here.
- `Pod.Services` — `ShellService`, `ConnectService`, `ShellApplicationService` translate domain types to/from these DTOs.
- `LeapVR.Shell.Services` (kiosk) — same usage on the client side.
