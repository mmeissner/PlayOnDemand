# LeapVR.Content.Creator.Language

> Localized string resources for the Content Creator. English (`en-US`) and Simplified Chinese (`zh-CN`).

## Purpose

A pure resource assembly. Holds the strongly-typed `Resources` class generated from `Resources.resx` plus per-culture overrides in `Resources.en-US.resx` and `Resources.zh-CN.resx`. Consumed both from XAML (via `WPFLocalizeExtension`'s `{lex:Loc ...}` markup) and from C# (e.g. `Resources.Global_ProductName`, `Resources.Validation_ErrorVBoxPackageOpenEdit`, the `LocalizedDescription` attributes on `PlatformType` enum values).

The project participates in the **Microsoft Multilingual App Toolkit (MAT)** workflow: XLIFF files under `MultilingualResources/` are the translator-facing artefacts and are imported back into the per-culture `.resx` files via the MAT MSBuild targets.

## Tech

- **Target framework:** .NET Framework 4.7.1, Library
- **Key NuGet packages:** none (pure resources)
- **Toolchain:** Microsoft Multilingual App Toolkit 4.0 (`Microsoft.Multilingual.ResxResources.targets`). The `MATPrerequisite` target emits a build warning if the toolkit is missing locally — this means the build still succeeds without MAT, you just won't get pseudo-loc warnings or XLIFF round-tripping.
- **Project references (in this repo):** none

## Responsibility

**It IS responsible for:**
- Storing every user-visible string used by the Creator wizard.
- Producing a single satellite-resource-friendly assembly so culture switching (`LocalizeDictionary.Instance.Culture = …`) works at runtime.

**It is NOT responsible for:**
- Picking the active culture — that's `ShellViewModel` (Creator project), driven by `ContentCreatorConfig.Language` (`"english"` or `"schinese"`).
- Any UI rendering.

## Public API surface

- `public class LeapVR.Content.Creator.Language.Resources` — auto-generated by `PublicResXFileCodeGenerator`. ~400 string properties; representative examples:
  - `Global_ProductName`, `Global_Warning`, `Global_Browse_Container`, `Global_Browse_AllFiles`, `Global_Select_Directory`
  - `BasicInfo_SelectLanguage`, `BasicInfo_TargetPlatform`
  - `DetailInfo_Category`, `DetailInfo_Description`, `DetailInfo_Image`, `DetailInfo_ExecutablePathNotRelative`
  - `Summary_Output_FileAlreadyExists`, `Validation_InvalidFilenameCharFormat`, `Validation_VBoxPackage_Edit_MsgBox_Error_Caption`
  - `Complete_Done`, `Complete_Error`, `Complete_ErrorMessage`
  - `Enum_Platform_LeapVR`, `Enum_Platform_Steam` (referenced by `LocalizedDescription` attributes on `PlatformType`)
- `public static System.Globalization.CultureInfo Culture { get; set; }` — overrides the resource lookup culture (used by Creator's language switch handlers).

## Internal structure

```
LeapVR.Content.Creator.Language/
├── Resources.resx                  Default / fallback strings (407 lines)
├── Resources.en-US.resx            English overrides (14 lines — minimal; defaults are already English)
├── Resources.zh-CN.resx            Simplified Chinese (302 lines)
├── Resources.Designer.cs           Auto-generated strongly-typed accessor
└── MultilingualResources/
    ├── LeapVR.Content.Creator.Language.en-US.xlf   MAT XLIFF for English
    └── LeapVR.Content.Creator.Language.zh-CN.xlf   MAT XLIFF for Chinese
```

## How it's used at the call site

From C# (typically inside view models or `LocalizedDescription` enum attributes):

```csharp
using LeapVR.Content.Creator.Language;

DisplayName = Resources.Global_ProductName;
MessageBox.Show(Resources.Validation_ErrorVBoxPackageOpenEdit,
                Resources.Validation_VBoxPackage_Edit_MsgBox_Error_Caption);
```

From XAML (via `WPFLocalizeExtension`'s `lex:Loc` markup):

```xml
xmlns:lex="http://wpflocalizeextension.codeplex.com"
lex:LocalizeDictionary.DesignCulture="en-US"
lex:ResxLocalizationProvider.DefaultAssembly="LeapVR.Content.Creator.Language"
lex:ResxLocalizationProvider.DefaultDictionary="Resources"

<TextBlock Text="{lex:Loc BasicInfo_TargetPlatform}" />
```

The two paths share the same `.resources` satellite assemblies — switching `LocalizeDictionary.Instance.Culture` updates XAML bindings instantly, and assigning `CultureInfo.DefaultThreadCurrentUICulture` makes the strongly-typed `Resources.X` accessor follow.

## Notable patterns / gotchas

- **Default resource generator is `PublicResXFileCodeGenerator`** (not the default `ResXFileCodeGenerator`). This is what makes `Resources` and its members `public` rather than `internal`, allowing references from `LeapVR.Content.Creator.Logic` (e.g. enum `LocalizedDescription` attributes on `PlatformType.LeapVr` / `PlatformType.Steam`).
- **Culture switching is process-wide and must be done in two places.** `ShellViewModel` does both `LocalizeDictionary.Instance.Culture = ...` (for XAML markup extension) and `CultureInfo.DefaultThreadCurrentUICulture = CultureInfo.DefaultThreadCurrentCulture = ...` (for `Resources.X` lookups in code). Both must be set; the XAML side does not propagate to the strongly-typed accessor.
- **Two languages, both stored in `ContentCreatorConfig.Language` as strings**: `"english"` or `"schinese"` (Steam-style identifiers). `ContentCreatorConfig.LanguageCountrycodeMap` further maps these to ISO country codes (`US`, `CN`) for the Steam Store API queries in `SteamApiDataAcquisition`.
- **`Resources.en-US.resx` is intentionally sparse** (~14 lines) — most English strings live in the neutral `Resources.resx` (~407 lines) and serve as the fallback. `en-US` only carries strings that need a US-regional variation. `zh-CN.resx` (~302 lines) is a near-full translation set.
- **`MATPrerequisite` target is non-fatal** — building on a machine without the Multilingual App Toolkit prints a warning but still succeeds; only XLIFF synchronization is skipped. CI machines do not need MAT installed.
- **`MultilingualFallbackLanguage` is `en`** (not `en-US`) — `.NET`'s resource fallback chain looks `zh-CN` → `zh` → `en` → neutral, so the neutral `Resources.resx` is what actually gets served when a Chinese key is missing.
- **Build configurations** match the rest of the tier (`Debug`, `Release`, `Release_ShellClient`, with x64 variants); MAT settings (`SuppressPseudoWarning` in Debug, `TranslationReport` in Release) toggle by configuration.
- **Adding a new string** = edit `Resources.resx` (the neutral file is the source of truth), let VS regenerate `Resources.Designer.cs`, then translate the new key in `Resources.zh-CN.resx`. The XLIFF files under `MultilingualResources/` are the translator hand-off format and are kept in sync by the MAT MSBuild targets when present.

## Consumers

- `LeapVR.Content.Creator` — both XAML (`{lex:Loc …}` markup) and C# (`Resources.X`) lookups across all wizard view models.
- `LeapVR.Content.Creator.Logic` — used by `Enums.cs` for the `[LocalizedDescription]` attributes on `PlatformType`.
